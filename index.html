網頁安全(Web-Security)是什麼？

泛指針對網頁、網站上的資訊安全，涵蓋程式漏洞、邏輯漏洞、資訊洩漏等。

XSS(Cross-Site-Scripting)

XSS 可中譯為跨網站指令碼攻擊，簡單介紹，就是「攻擊者」運用一些小技巧，將 Javascript或HTML的 element 置入在別人的網站中，因為HTML與Javascript的互動十分密切，甚至可以利用 Javascript 來連結到外部網站，或者用「受害者」的身分來傳輸資料，這會使得本來「受害者」以為安全的網站變得十分危險。

只要你打完帳號密碼(也就是那兩個框)，按submit後，就會將密碼轉送到一個惡意的server，也就是說，假如 attacker 發現可以插入代碼的漏洞，那麼他就可以以「 document.getElementById(‘id’).onfocusout = function() {…} 」之類的方法動態插入這個 <script>。

目前 XSS 攻擊的種類大致可以分成以下幾種類型：

  • Stored XSS (儲存型)
  • Reflected XSS (反射型)
  • DOM-Based XSS (基於 DOM 的類型)

Stored XSS (儲存型)

會被保存在伺服器資料庫的惡意Javascript代碼稱為 Stroed XSS，常見在文章的留言板，因為留言會被存進資料庫，如果裡面有惡意 Javascript 代碼，就會被存起來。

Reflected XSS (反射型)

不會被保存在伺服器，通常是藉由 GET 這個類型的request，使得代碼直接嵌入網頁後端，舉一個php的實例:

DOM 全名為 Document Object Model，用以描述 HTML 文件的表示法，它讓我們可以使用 JavaScript 動態產生完整的網頁，而不必透過伺服器，點以下連結來深入了解

從沒有Ubuntu開始安裝Ubuntu

tags: 奇夆的雙系統下載教學

在Windows10裡面要做的事

Windows10，在「Windows設定」搜尋「電源與睡眠」，點選「其他電源設定」，點選「選擇按下電源按鈕時的行為」，點選「變更目前無法使用的設定」，取消勾選「開啟快速啟動」。

打開媒體櫃，在「本機」上面按右鍵，點選「管理」，點選「存放裝置」裡面的「磁碟管理」，然後在想要壓縮的磁碟分割區上面點右鍵，點選「壓縮磁碟區」。

  • 如果要把Ubuntu的根目錄和家目錄分開來的話，建議壓縮SSD空間來給根目錄，然後HDD給家目錄。根目錄建議有20GB以上，家目錄就看自己打算放多少檔案。
  • 如果看不懂上面的東西，建議就切一塊30GB以上的空間給Ubuntu就好。

切好磁碟區之後，就重新開機，進BIOS。

疑難排解

  • Acer有些電腦在裝好之後沒辦法抓到Ubuntu的Grub來開機，解決方法是把Secure Boot開啟，然後把Ubuntu的EFI檔案設定為trusted。操作請參考 https://askubuntu.com/questions/597213/bootable-device-not-found-after-clean-install-of-ubuntu-14-04-uefi-on-acer

  • Boot Repair
    https://help.ubuntu.com/community/Boot-Repair
    一鍵修復的懶人工具，用用來安裝的USB開機，進到「Try Ubuntu Without Installing」，用Firefox搜尋Boot Repair，依照指示安裝好，再依照指示按修復，跑好之後重新開機，看看問題有沒有解決。
