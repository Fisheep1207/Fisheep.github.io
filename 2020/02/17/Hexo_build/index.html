安裝 Hexo

官方教學：https://hexo.io/zh-tw/docs/

因為有官方教學，這裡我簡單帶過

  • 在Ubuntu中，使用以下command安裝
  • 安裝完Hexo後，就在桌面創建一個資料夾，並在資料夾中執行Hexo的指令 看到資料夾變成這樣的架構就代表完成了：

    簡單介紹各個檔案的功用：
  • _config.yml：網站 配置 檔案，可以在此配置大部分的設定。
  • package.json：套件版本控制
  • scaffolds：鷹架 資料夾。建立新文章時，會根據 scaffold 來建立檔案。
  • source：原始檔案資料夾是放置內容的地方。（後來要新增其他分頁例如：tags、categories…等等，也是在這裡新增）。
  • themes：主題 資料夾。Hexo 會根據主題來產生靜態檔案。

其實到這邊，再執行完兩個指令就可以在本地端看到自己的網站！但是在那之前先讓我介紹兩個重要的指令

在 GitHub 上新增 GitHub Pages

這裡就到了最後一步——讓大家看到你的網站！
我們使用GitPage來搭建就是因為它最容易也最好懂！

  1. 首先在 Github 上面創建一個新的專案，名稱叫作

    .github.io

    例如： Fisheep1207.github.io

    貼心小提醒：

    • 這裡要記得，<username>一定要跟你 github 的 username一樣喔！不然 github page 會找不到你的資料在哪裡。
    • github 的 private 專案要錢，如果不付錢的話，就乖乖選用 public 專案喔！

  2. 接下來，回到 command line ， 先安裝 deploy git 插件

  3. 進入 /Blog/_config.yml

  4. 使用

  5. 上傳完畢之後，就可以上

    .github.io 看到你自己的網站摟！

    貼心小提醒：

  • 當你發現你的網站在本地端可以正常運作，也確定已經上傳到github，但是你按 F5 重新整理網頁時，仍然看不到變化，那有可能是你的瀏覽器紀錄了一些 cache ，導致你一直訪問到之前的網頁，這個時候要按 Ctrl+F5，告訴瀏覽器：「無論你有沒有記下任何東西，我要你全部重新存取！！」，這樣才會看到改變喔！！

  • 你可能會上網查詢一些微幅修改.css，來讓介面變得更美的方式，你會發現，奇怪，明明已經 $hexo g -d$了，怎麼看不到改變，原因可能是你沒有操作

    這樣才會讓你的全部文件確實的更動，整體的頁面也才會有改變喔！但是public文件會被全部刪除，所以如果你有手動在裏面存圖片或是其他文件，要記得自己做了哪些更動喔！
